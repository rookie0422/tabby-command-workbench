# 项目上下文

本文件是给后续 agent 使用的轻量项目上下文，只记录当前仍然有效、长期有用的信息。只有需要追溯历史原因时，才查看 Git 历史或旧 Codex 会话。

## 产品意图

`tabby-command-workbench` 是一个 Tabby 终端插件，提供右侧命令工作台。它面向串口、ADB、Linux 和项目调试等高频终端工作。

顶部 Tab 是用户自定义场景，不是功能页切换。每个场景都包含同样的三块区域：

- 快捷命令按钮：用于快速填充或复制。
- 常用命令：命令正文是主视觉，左键复制。
- 大型草稿区：每个场景独立，用户可以粘贴、编辑、选中和复制任意多行文本。

主界面应该像工具面板，而不是后台管理页。左键负责高频动作；新增、编辑、删除、改色、填充和发送等管理动作放在右键菜单或弹窗里。

## 当前行为规则

- 快捷按钮只显示用户配置的按钮名称。
- 快捷按钮可以是填充、复制或发送并回车。会发送并回车的按钮必须在 UI 上和仅填充按钮区分开。
- 快捷按钮内容支持 `{{name}}` 参数占位符；执行前统一收集参数，并复用到整段命令。
- 自动回车模式下，多行内容按行顺序发送；`{{delay:2000}}` 或 `{{delay:2s}}` 表示简单定时等待。delay 只在直接执行模式生效，编辑器插入或保存合法 delay 时会自动启用直接执行。该能力只按时间发送，不判断终端输出、命令成功与否，也不做条件分支。
- 命中高风险命令的自动发送路径需要安全边界：按钮保存或首次执行时确认；仅填充不弹确认。
- 快捷按钮区保持紧凑：根据宽度使用三列或四列，最多显示三排，更多按钮在该区域内部滚动。
- 常用命令优先突出命令正文，不强调说明文本；最多显示约五条，超出后在该区域内部滚动。
- 草稿区占用侧栏剩余高度，表现为普通的大型文本框；原生复制、粘贴、选中和撤销必须可用。
- 顶部场景 Tab 支持拖动排序；拖动过程中其他 Tab 平滑让位，只有在 Tab 区域内有效放下时才持久化新顺序，取消拖动恢复原顺序。
- 侧栏采用停靠式布局：展开时给 Tabby 主界面让出右侧宽度，尽量压缩终端主体而不是遮挡终端内容。
- 侧栏左边缘可拖动调整宽度，宽度写入 `sidebarWidth` 并持久化。
- 编辑表单使用窗口级弹窗，避免编辑时挤压列表布局或改变滚动高度。
- 执行动作后，如预期需要继续操作终端，应把焦点交回活动终端。

## 实现地图

- `src/index.ts`：连接 Angular 模块、设置页 provider、配置 provider 和终端装饰器。
- `src/terminalDecorator.ts`：把活动终端 Tab 注册到 `CommandWorkbenchService`。
- `src/quickButtons.service.ts`：负责 DOM 侧栏和大部分运行时行为。
- `src/commandSequence.ts`：负责参数占位符、命令序列、delay 和危险命令扫描。此类纯逻辑优先放在独立模块并补测试，不继续塞进 service。
- `src/model.ts`：创建默认数据并归一化持久化数据。
- `src/config.ts`：在当前配置和旧配置之间选择真正应该使用的持久化数据。
- `src/configProvider.ts`：声明当前和旧版顶层配置键，确保 Tabby 启动迁移期间仍能看到旧数据。
- `src/settingsTab.component.ts` 和 `src/settingsTabProvider.ts`：在 Tabby 中暴露插件设置。

## 配置与迁移

当前配置键是 `commandWorkbench`。旧配置键 `serialCommandSidebar` 只作为迁移来源保留。

Tabby 可能在服务运行前创建空结构默认值，所以空的 `commandWorkbench: {}` 不能优先于真实旧数据。`src/config.ts` 中的选择逻辑负责处理这个情况。

Tabby 配置持久化基于代理对象。不要假设整对象赋值一定能保存；应更新需要持久化的具体模型字段。当前集中写入路径是 `CommandWorkbenchService.writeModelToConfig`。

当前归一化配置版本是 `2`，分类中保存 `quickButtons`、`commonCommands` 和 `scratchpad`。

## 验证方式

- `npm test`：验证配置归一化和迁移选择。
- `npm run typecheck`：验证 TypeScript 类型。
- `npm run build`：构建 Tabby 插件包。
- `npm run check`：发布前或大范围重构前运行完整本地检查。

修改 UI 行为时，静态检查不够；条件允许时还要在 Tabby 中验证相关交互，尤其是场景切换、右键菜单、弹窗编辑、命令填充/复制行为，以及草稿区的原生剪贴板快捷键。

停靠式侧栏依赖 DOM/CSS 让 Tabby 主界面为右侧面板留出空间，并在宽度变化后触发布局刷新。修改这部分时，还要验证展开/收起是否恢复主界面宽度、拖拽宽度是否持久化，以及终端列宽是否重新计算。
