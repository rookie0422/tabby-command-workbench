# 更新日志

## v1.2.2 - 2026-06-22

### 新增

- 场景 Tab、快捷按钮和常用命令支持跟手拖动排序和平滑让位，顺序自动持久化。

### 修复

- 完善不同尺寸元素、首尾边界和快捷按钮末行空缺时的换位判定。
- 避免拖动过程中出现无效空槽、额外滑条、误触操作或滚动位置重置。

## v1.2.1 - 2026-06-18

### 修复

- 修复 Tabby 标签页位置设为左侧或右侧时，命令工作台展开后终端内容区被错误挤压、只剩标签栏可见的问题。

### 文档

- README 增加平台兼容性说明：插件设计上适用于 Windows、macOS 和 Linux，但当前主要在 Windows 上进行日常验证。

## v1.2.0 - 2026-06-18

### 新增

- 快捷按钮支持 `{{name}}` 参数占位符，点击时通过弹窗填写参数，并复用到整段命令。
- 自动回车模式支持多行顺序发送；可用 `{{delay:2000}}` 或 `{{delay:2s}}` 插入简单等待。
- 编辑快捷按钮时可一键插入参数占位符和 delay 片段；插入 delay 会自动启用直接执行。
- 自动发送按钮增加“填充 / 发送 / 复制”的可见模式标记，高风险自动发送按钮使用危险样式。
- 高风险自动发送命令增加一次性确认边界：保存危险按钮或首次执行未确认按钮时提醒，确认后不再反复打断。
- README 增加产品界面预览图和参数化多行命令示例。

### 修复

- 设置页保存时先归一化配置，再写回完整模型字段，避免只保存部分字段造成配置不完整。
- 多行仅填充优先走 Tabby 粘贴路径，避免部分 shell 直接 `sendInput` 多行文本时出现顺序异常。
- 仅填充时会剥离合法的 delay 控制行，避免把 `{{delay:...}}` 直接写进终端。

### 变更

- 命令参数、delay 序列和危险命令扫描抽到 `src/commandSequence.ts`，降低 `quickButtons.service.ts` 的继续膨胀风险。
- 发布 sourcemap 改为 `nosources-source-map`，不再把完整 `sourcesContent` 放入 npm 包。
- README 增加安全提示：命令和草稿会持久化到 Tabby 配置中，不要保存 token、密码或生产密钥。

## v1.1.1 - 2026-06-17

### 修复

- 增强右键菜单关闭和清理逻辑：记录监听器和计时器引用，在 `closeContextMenu` 中显式移除，避免事件泄漏。
- 设置页 `save()` 现在通过 `normalizeConfig` 填充缺失字段后再写回 store，避免只更新两个字段导致配置不完整。
- resize 处理增加 `pointercancel`、`blur` 和 `lostpointercapture` 清理路径，防止拖动异常中断时状态卡死。

### 新增

- 常用命令卡片现在支持显示 `description` 副标题，方便区分用途相近的命令。

### 变更

- 持久化时剥离 `tempSnippets` 字段，避免死数据写入配置存储。
- 提取常量 `constants.ts` 和样式字符串 `styles.ts`，减少 service 文件中的魔数和内联 CSS。
- 扩展测试覆盖：配置归一化边界、tempSnippets → scratchpad 迁移、sidebarWidth 范围钳位。

## v1.1.0 - 2026-06-16

### 新增

- 命令工作台改为右侧停靠布局，展开时为 Tabby 主内容区预留右侧空间，而不是直接遮挡终端内容。
- 工作台左边缘支持拖动调整宽度，宽度会写入 `sidebarWidth` 并持久化。
- 新增轻量项目上下文文件：`AGENTS.md` 和 `docs/project-context.md`，方便后续维护。

### 变更

- 展开工作台时，Tabby 顶部工具栏保持完整宽度，只让活动内容区为工作台让位。
- 移除设置页里的宽度滑块，侧栏宽度改为直接在主界面拖动调整。
- 更新 README 中关于停靠侧栏和拖动改宽的说明。

### 修复

- 修复工作台展开后 Tabby 设置按钮被挤出可视区域的问题。
- 修复终端内容仍被侧栏遮挡的问题：改为压缩 Tabby 内层内容容器，并在宽度变化后触发终端布局刷新。
